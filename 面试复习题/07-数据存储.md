## 一、浏览器存储

### 1. 存储方式对比
| 特性 | Cookie | localStorage | sessionStorage | IndexedDB |
|------|--------|-------------|---------------|-----------|
| 容量 | 4KB | 5MB | 5MB | 几百MB+ |
| 请求携带 | 自动携带 | 不携带 | 不携带 | 不携带 |
| 生命周期 | 可设过期时间 | 永久 | 标签页关闭 | 永久 |
| 作用域 | 同源+路径 | 同源 | 同源+标签页 | 同源 |
| API | document.cookie | 同步 API | 同步 API | 异步 API |
| 存储类型 | 字符串 | 字符串 | 字符串 | 结构化数据 |
| Web Worker | 可访问 | 可访问 | 可访问 | 可访问 |

### 2. Cookie
```javascript
// 设置 Cookie
document.cookie = 'name=Tom; max-age=3600; path=/; secure; SameSite=Strict';

// Cookie 属性
// domain    — 域名（默认当前域）
// path      — 路径（默认 /）
// max-age   — 过期时间（秒）
// expires   — 过期时间（UTC 日期）
// secure    — 仅 HTTPS
// httpOnly  — JS 无法读取（防 XSS）
// SameSite  — 跨站策略（Strict/Lax/None）

// 读取（返回所有 cookie 的字符串）
document.cookie; // "name=Tom; age=25"

// 第三方 Cookie 限制
// Chrome 已逐步淘汰第三方 Cookie
// 替代方案: CHIPS (Cookies Having Independent Partitioned State)
```

### 3. localStorage / sessionStorage
```javascript
// 基本操作
localStorage.setItem('key', 'value');
localStorage.getItem('key');
localStorage.removeItem('key');
localStorage.clear();

// 存储对象需要序列化
localStorage.setItem('user', JSON.stringify({ name: 'Tom' }));
const user = JSON.parse(localStorage.getItem('user'));

// 监听跨标签页变化（同源）
window.addEventListener('storage', (e) => {
  console.log(e.key, e.oldValue, e.newValue);
});
```

### 4. IndexedDB
```javascript
// 打开数据库
const request = indexedDB.open('myDatabase', 1);

request.onupgradeneeded = (e) => {
  const db = e.target.result;
  if (!db.objectStoreNames.contains('users')) {
    const store = db.createObjectStore('users', { keyPath: 'id' });
    store.createIndex('name', 'name', { unique: false });
    store.createIndex('email', 'email', { unique: true });
  }
};

request.onsuccess = (e) => {
  const db = e.target.result;

  // 写入数据
  const tx = db.transaction('users', 'readwrite');
  const store = tx.objectStore('users');
  store.add({ id: 1, name: 'Tom', email: 'tom@example.com' });

  // 查询数据
  const getRequest = store.get(1);
  getRequest.onsuccess = () => console.log(getRequest.result);

  // 使用索引查询
  const index = store.index('name');
  const nameRequest = index.getAll('Tom');

  // 使用游标遍历
  const cursorRequest = store.openCursor();
  cursorRequest.onsuccess = (e) => {
    const cursor = e.target.result;
    if (cursor) {
      console.log(cursor.value);
      cursor.continue();
    }
  };
};

// 推荐使用封装库: idb / Dexie.js
import { openDB } from 'idb';
const db = await openDB('myDatabase', 1, {
  upgrade(db) {
    db.createObjectStore('users', { keyPath: 'id' });
  },
});
await db.put('users', { id: 1, name: 'Tom' });
const user = await db.get('users', 1);
```

## 二、HTTP 缓存

### 1. 强缓存
浏览器直接使用本地缓存，不发送请求到服务器。

```http
// 方式1: Cache-Control（优先级更高）
Cache-Control: max-age=3600          // 缓存 1 小时
Cache-Control: no-cache              // 跳过强缓存，走协商缓存
Cache-Control: no-store              // 完全不缓存
Cache-Control: public                // 允许中间代理缓存
Cache-Control: private               // 仅浏览器缓存
Cache-Control: immutable             // 资源永不过期

// 方式2: Expires（HTTP/1.0，已过时）
Expires: Wed, 21 Oct 2025 07:28:00 GMT
```

### 2. 协商缓存
强缓存过期后，浏览器向服务器验证资源是否更新。

```http
// 方式1: ETag + If-None-Match（优先级更高）
// 响应头
ETag: "abc123"
// 下次请求头
If-None-Match: "abc123"
// 如果未修改 → 304 Not Modified（不传 body）

// 方式2: Last-Modified + If-Modified-Since
// 响应头
Last-Modified: Wed, 21 Oct 2025 07:28:00 GMT
// 下次请求头
If-Modified-Since: Wed, 21 Oct 2025 07:28:00 GMT
```

### 3. 缓存决策流程
```
请求资源
  ↓
有强缓存且未过期？ → Yes → 200 (from cache)
  ↓ No
有协商缓存？ → 发送验证请求
  ↓
资源未修改？ → Yes → 304 Not Modified
  ↓ No
200 + 新资源 + 新缓存头
```

### 4. 前端缓存策略实践
```nginx
# Nginx 配置示例
# HTML: 不缓存或短缓存（入口文件）
location / {
  add_header Cache-Control "no-cache";
}

# 静态资源: 长缓存（文件名带 hash）
location /assets/ {
  add_header Cache-Control "public, max-age=31536000, immutable";
}
```

## 三、Service Worker & PWA

### 1. Service Worker 生命周期
```
Register → Install → Waiting → Activate → Fetch
                        ↑           ↓
                    skipWaiting  claim
```

### 2. 离线缓存
```javascript
// 注册 Service Worker
navigator.serviceWorker.register('/sw.js');

// sw.js
const CACHE_NAME = 'v1';
const CACHE_URLS = ['/', '/index.html', '/styles.css', '/app.js'];

self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME).then(cache => cache.addAll(CACHE_URLS))
  );
});

self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request).then(response => {
      return response || fetch(event.request).then(fetchResponse => {
        // 缓存新请求
        return caches.open(CACHE_NAME).then(cache => {
          cache.put(event.request, fetchResponse.clone());
          return fetchResponse;
        });
      });
    })
  );
});

self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then(keys =>
      Promise.all(keys.filter(key => key !== CACHE_NAME).map(key => caches.delete(key)))
    )
  );
});
```

### 3. 缓存策略
| 策略 | 说明 | 适用场景 |
|------|------|---------|
| Cache First | 优先缓存 | 静态资源 |
| Network First | 优先网络 | API 请求 |
| Stale While Revalidate | 先缓存再后台更新 | 非关键请求 |
| Network Only | 仅网络 | 非 GET 请求 |
| Cache Only | 仅缓存 | 离线页面 |

## 四、前端数据管理

### 1. Token 存储
```javascript
// 方案对比
// localStorage: 简单，但有 XSS 风险
// Cookie (httpOnly): 防 XSS，但有 CSRF 风险
// memory: 最安全，但刷新丢失

// 最佳实践: 双 Token 方案
// Access Token → 内存（或短期 Cookie）
// Refresh Token → httpOnly Cookie

// Token 刷新机制
let accessToken = '';

async function fetchWithAuth(url, options = {}) {
  if (isTokenExpired(accessToken)) {
    const { token } = await refreshToken();
    accessToken = token;
  }
  return fetch(url, {
    ...options,
    headers: { ...options.headers, Authorization: `Bearer ${accessToken}` },
  });
}
```

### 2. 离线数据同步
```javascript
// Background Sync API
navigator.serviceWorker.ready.then(registration => {
  registration.sync.register('sync-data');
});

// sw.js
self.addEventListener('sync', (event) => {
  if (event.tag === 'sync-data') {
    event.waitUntil(syncOfflineData());
  }
});
```

## 六、数据存储面试高频题

### Q1: Cookie、localStorage、sessionStorage 区别？
- Cookie: 4KB，请求自动携带，可设过期时间
- localStorage: 5MB，不携带，永久存储
- sessionStorage: 5MB，不携带，标签页关闭清除

### Q2: Token 存哪里？
推荐双 Token：Access Token 存内存，Refresh Token 存 httpOnly Cookie。兼顾安全和体验。

### Q3: IndexedDB 什么时候用？
- 大量结构化数据存储
- 离线应用
- 需要事务支持
- 存储二进制数据（Blob）

### Q4: HTTP 缓存头怎么配？
- HTML: `Cache-Control: no-cache`（每次验证）
- JS/CSS/图片（带 hash）: `Cache-Control: max-age=31536000, immutable`
- API: `Cache-Control: no-store` 或短时间缓存

### Q5: 如何实现离线应用？
1. Service Worker 拦截请求
2. Cache API 缓存资源
3. IndexedDB 存储数据
4. Background Sync 后台同步
5. Web App Manifest 添加到桌面
