## 一、HTTP 协议

### 1. HTTP/1.1 → HTTP/2 → HTTP/3 演进
| 特性 | HTTP/1.1 | HTTP/2 | HTTP/3 |
|------|----------|--------|--------|
| 传输层 | TCP | TCP | QUIC (UDP) |
| 多路复用 | ❌ 队头阻塞 | ✅ 同一连接并行 | ✅ 无队头阻塞 |
| 头部压缩 | ❌ | ✅ HPACK | ✅ QPACK |
| 服务端推送 | ❌ | ✅ | ✅ |
| 连接建立 | 1 RTT | 1 RTT (TLS: 2-3) | 0-1 RTT |
| 丢包影响 | - | 所有流阻塞 | 仅影响单个流 |

### 2. HTTPS
```
HTTP + TLS = HTTPS

TLS 握手过程:
1. Client Hello — 支持的 TLS 版本、加密套件、随机数
2. Server Hello — 选定的加密套件、证书、随机数
3. 密钥交换 — ECDHE 等算法生成对称密钥
4. 加密通信 — 使用对称密钥加密数据

证书验证链: 浏览器信任根 CA → 中间 CA → 服务器证书
```

### 3. TCP 三次握手 & 四次挥手
```
三次握手（建立连接）:
Client → SYN → Server
Client ← SYN+ACK ← Server
Client → ACK → Server

四次挥手（断开连接）:
Client → FIN → Server
Client ← ACK ← Server
Client ← FIN ← Server
Client → ACK → Server
```

### 4. 常见 HTTP 状态码
```
200 OK              — 请求成功
301 Moved Permanently — 永久重定向（SEO 权重转移）
302 Found           — 临时重定向
304 Not Modified    — 协商缓存命中
400 Bad Request     — 请求参数错误
401 Unauthorized    — 未认证
403 Forbidden       — 无权限
404 Not Found       — 资源不存在
429 Too Many Requests — 请求过多
500 Internal Server Error — 服务器错误
502 Bad Gateway     — 网关/上游错误
503 Service Unavailable — 服务不可用
```

### 5. GET vs POST
| 维度 | GET | POST |
|------|-----|------|
| 参数位置 | URL 查询字符串 | 请求体 |
| 长度限制 | 浏览器有限制（~2KB） | 无限制 |
| 幂等性 | 幂等 | 非幂等 |
| 缓存 | 可被缓存 | 不缓存 |
| 安全性 | 参数暴露在 URL | 相对安全 |
| 书签 | 可收藏 | 不可收藏 |

### 7. WebSocket
```javascript
// 全双工通信，适用于实时场景
const ws = new WebSocket('wss://example.com/ws');

ws.onopen = () => ws.send(JSON.stringify({ type: 'message', data: 'hello' }));
ws.onmessage = (e) => console.log(JSON.parse(e.data));
ws.onclose = () => console.log('disconnected');

// 心跳保活
setInterval(() => {
  if (ws.readyState === WebSocket.OPEN) ws.send(JSON.stringify({ type: 'ping' }));
}, 30000);
```

## 二、前端安全

### 1. XSS（跨站脚本攻击）
```javascript
// 反射型 XSS: URL 参数中注入脚本
// https://example.com/search?q=<script>alert(document.cookie)</script>

// 存储型 XSS: 恶意脚本存入数据库
// 评论: <img src=x onerror="alert(document.cookie)">

// DOM 型 XSS: 前端直接操作 DOM
// document.getElementById('output').innerHTML = userInput;

// 防御:
// 1. 输出转义
function escapeHTML(str) {
  return str.replace(/[&<>"']/g, (c) => ({
    '&': '&amp;', '<': '&lt;', '>': '&gt;', '"': '&quot;', "'": '&#39;'
  })[c]);
}

// 2. CSP (Content Security Policy)
Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'

// 3. HttpOnly Cookie
// 4. 使用 textContent 代替 innerHTML
// 5. 框架默认转义（React/Vue）
```

```javascript
// 1. SameSite Cookie
Set-Cookie: token=xxx; SameSite=Strict

// 2. CSRF Token
// 服务端生成随机 token，表单提交时携带

// 3. 验证 Referer/Origin 头
// 4. 关键操作二次确认（验证码/密码）
```

## 三、性能优化

### 1. 加载性能
```
资源优化:
  ├── 代码分割（路由级、组件级）
  ├── Tree-shaking（移除未使用代码）
  ├── 压缩（Terser/SWC 压缩 JS，cssnano 压缩 CSS）
  ├── 图片优化（WebP/AVIF、懒加载、响应式图片）
  ├── 字体优化（font-display: swap、子集化）
  └── Gzip/Brotli 压缩

传输优化:
  ├── CDN 加速
  ├── HTTP/2 多路复用
  ├── preload / prefetch / preconnect
  ├── Service Worker 缓存
  └── DNS 预解析
```

### 2. 资源提示
```html
<!-- preload: 当前页面立即需要 -->
<link rel="preload" href="/font.woff2" as="font" crossorigin>

<!-- prefetch: 下一页面可能需要 -->
<link rel="prefetch" href="/next-page.js">

<!-- preconnect: 提前建立连接 -->
<link rel="preconnect" href="https://api.example.com">

<!-- dns-prefetch: 仅 DNS 解析 -->
<link rel="dns-prefetch" href="https://cdn.example.com">
```

### 3. 渲染性能
```javascript
// 1. 减少 DOM 操作
// 批量操作: DocumentFragment / 虚拟 DOM
const fragment = document.createDocumentFragment();
items.forEach(item => fragment.appendChild(createElement(item)));
container.appendChild(fragment);

// 2. 避免强制同步布局（读取和写入交替）
// Bad: 读→写→读→写（每次读都可能触发重排）
// Good: 读→读→写→写

// 3. 使用 CSS will-change 提升层
// .element { will-change: transform; }

// 4. 虚拟列表
import { FixedSizeList } from 'react-window';
<FixedSizeList height={600} itemCount={10000} itemSize={50}>
  {({ index, style }) => <div style={style}>{items[index]}</div>}
</FixedSizeList>

// 5. Web Worker 处理耗时计算
const worker = new Worker('/heavy-computation.js');
worker.postMessage(data);
worker.onmessage = (e) => console.log(e.data);
```

## 四、网络与性能面试高频题

### Q1: 从输入 URL 到页面显示的过程？
```
1. DNS 解析 — 域名 → IP（递归查询，有 DNS 缓存）
2. TCP 连接 — 三次握手
3. TLS 握手 — HTTPS 连接
4. HTTP 请求 — 发送请求报文
5. 服务器处理 — 返回响应
6. 浏览器解析:
   a. 解析 HTML → DOM Tree
   b. 解析 CSS → CSSOM Tree
   c. 合并 → Render Tree
   d. Layout（布局计算）
   e. Paint（绘制）
   f. Composite（合成层）
7. 执行 JS（阻塞 HTML 解析）
8. 触发 DOMContentLoaded / load 事件
```

### Q2: 重排（Reflow）vs 重绘（Repaint）？
- **重排**: 元素几何属性变化（尺寸、位置）→ 重新计算布局
- **重绘**: 外观变化（颜色、背景）→ 不影响布局
- **触发重排**: width/height/padding/margin/display/position
- **仅重绘**: color/background/visibility/box-shadow
- **优化**: 批量修改样式、使用 transform/opacity（GPU 加速）

### Q3: 为什么操作 DOM 慢？
- DOM 在渲染引擎，JS 在 JS 引擎，跨引擎通信有开销
- DOM 操作可能触发重排/重绘
- 现代框架用虚拟 DOM 减少直接 DOM 操作

### Q4: HTTP/2 多路复用解决了什么问题？
HTTP/1.1 的队头阻塞：一个 TCP 连接同时只能处理一个请求。HTTP/2 将请求拆分为帧（Frame），多个请求的帧可以在同一 TCP 连接上交错传输。

### Q5: CORS 预检请求什么时候触发？
非简单请求触发预检：
- 方法不是 GET/POST/HEAD
- Content-Type 不是 text/plain、multipart/form-data、application/x-www-form-urlencoded
- 自定义 Header
