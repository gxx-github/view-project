## 一、HTML

### 1. HTML5 新特性有哪些？
- 语义化标签：`<header>` `<nav>` `<section>` `<article>` `<aside>` `<footer>`
- 多媒体：`<audio>` `<video>` `<canvas>`
- 表单增强：`date` `email` `range` `number` `placeholder` `required`
- 本地存储：`localStorage` `sessionStorage` `indexedDB`
- 新 API：Geolocation、Web Worker、WebSocket、Drag & Drop、Web Speech（见下方详解）

### 2. 语义化标签的意义？
- 提升可访问性（屏幕阅读器）
- 有利于 SEO 搜索引擎抓取
- 代码可读性更强，结构清晰
- 便于团队维护

### 3. script 标签的 defer 和 async 区别？
| 特性 | defer | async |
|------|-------|-------|
| 执行时机 | DOM 解析完后，DOMContentLoaded 前 | 下载完立即执行 |
| 执行顺序 | 按声明顺序执行 | 谁先下载完谁先执行 |
| 阻塞 DOM | 不阻塞 | 执行时阻塞 |
| 使用场景 | 依赖 DOM 的脚本 | 独立脚本如统计/广告 |

### 4. HTML 渲染过程？
```
HTML → DOM Tree
CSS  → CSSOM Tree
     ↘ Render Tree → Layout → Paint → Composite
```

### 5. meta 标签常用属性？
```html

<meta name="description" content="页面描述">
```

## 二、HTML5 核心 API 详解

### 1. Geolocation（地理定位）

**是什么：** 浏览器提供的获取用户地理位置的 API，基于 GPS、WiFi、IP 等方式定位。

**核心方法：**
```javascript
// 获取当前位置（一次性）
navigator.geolocation.getCurrentPosition(
  (position) => {
    console.log('纬度:', position.coords.latitude);    // 如 39.9042
    console.log('经度:', position.coords.longitude);   // 如 116.4074
    console.log('精度:', position.coords.accuracy);    // 单位: 米
    console.log('海拔:', position.coords.altitude);    // 可能为 null
    console.log('速度:', position.coords.speed);       // 可能为 null
    console.log('时间戳:', position.timestamp);
  },
  (error) => {
    switch (error.code) {
      case error.PERMISSION_DENIED:   // 用户拒绝了定位权限
        console.error('用户拒绝了定位请求');
        break;
      case error.POSITION_UNAVAILABLE: // 位置信息不可用
        console.error('位置信息不可用');
        break;
      case error.TIMEOUT:              // 请求超时
        console.error('请求超时');
        break;
    }
  },
  {
    enableHighAccuracy: true,  // 高精度定位（更耗电）
    timeout: 5000,             // 超时时间 ms
    maximumAge: 0,             // 不使用缓存位置
  }
);

// 持续监听位置变化（实时追踪）
const watchId = navigator.geolocation.watchPosition(
  (position) => { console.log(position.coords); },
  (error) => { console.error(error); },
  { enableHighAccuracy: true }
);

// 停止监听
navigator.geolocation.clearWatch(watchId);
```

**注意事项：**
- 必须在 HTTPS 环境下使用（localhost 除外）
- 首次调用会弹出权限确认框，用户可以拒绝
- 在手机上精度更高（有 GPS），PC 端精度较低（基于 IP/WiFi）
- 隐私合规：需告知用户为什么需要定位

**实际应用：** 外卖配送定位、运动轨迹记录、附近店铺搜索、天气查询

### 2. Web Worker（多线程）

**是什么：** 在浏览器后台创建独立线程执行 JS 代码，不阻塞主线程（UI 线程）。

**为什么需要：** JS 是单线程的，耗时计算（大数据处理、图片处理、加密）会阻塞 UI 导致卡顿。Web Worker 把这些任务放到后台线程。

**主线程代码：**
```javascript
// 创建 Worker
const worker = new Worker('heavy-task.js');

// 也可以用 Blob URL 内联创建（不需要单独文件）
const workerCode = `
  self.onmessage = function(e) {
    const result = heavyComputation(e.data);
    self.postMessage(result);
  }
  function heavyComputation(data) {
    // 耗时计算
    let sum = 0;
    for (let i = 0; i < data; i++) sum += i;
    return sum;
  }
`;
const blob = new Blob([workerCode], { type: 'application/javascript' });
const worker = new Worker(URL.createObjectURL(blob));

// 发送数据给 Worker
worker.postMessage(100000000);

// 接收 Worker 返回的结果
worker.onmessage = (e) => {
  console.log('Worker 计算结果:', e.data);
  worker.terminate(); // 用完后关闭 Worker，释放资源
};

// 错误处理
worker.onerror = (e) => {
  console.error('Worker 错误:', e.message);
};
```

**Worker 线程代码（heavy-task.js）：**
```javascript
// Worker 内部无法访问: DOM、window、document
// Worker 内部可以访问: fetch、IndexedDB、setTimeout、importScripts

self.onmessage = function(e) {
  const data = e.data;

  // 执行耗时任务
  const result = doHeavyWork(data);

  // 把结果发回主线程
  self.postMessage(result);
};

// 可以加载其他脚本
importScripts('utils.js');
```

**Worker 类型：**

| 类型 | 说明 |
|------|------|
| Dedicated Worker | 专用 Worker，只能被创建它的页面使用 |
| Shared Worker | 共享 Worker，可被同源多个页面共享 |
| Service Worker | 特殊 Worker，拦截网络请求、离线缓存（PWA 核心） |

**通信进阶 — Transferable Objects：**
```javascript
// 大数据传输（零拷贝，直接转移所有权）
const arrayBuffer = new ArrayBuffer(1024 * 1024 * 100); // 100MB
worker.postMessage(arrayBuffer, [arrayBuffer]); // 第二个参数指定要转移的对象
// 转移后主线程不能再访问这个 arrayBuffer（所有权交给了 Worker）
```

**实际应用：** 大数据排序/过滤、图片压缩（Canvas 像素处理）、加密解密、PDF 解析、虚拟列表数据预处理

### 3. WebSocket（全双工通信）

**是什么：** 基于 TCP 的全双工通信协议，浏览器和服务器可以互相主动推送数据，不需要像 HTTP 那样每次都由客户端发起。

**HTTP vs WebSocket：**
```
HTTP: 客户端请求 → 服务器响应 → 断开（每次都要重新建立连接）
HTTP 长轮询: 客户端请求 → 服务器hold住 → 有数据才响应 → 客户端再请求（模拟双向）
WebSocket: 建立连接（一次握手）→ 双方随时互发数据 → 持续保持
```

**基础使用：**
```javascript
// 创建连接（ws:// 或 wss:// 加密）
const ws = new WebSocket('wss://example.com/chat');

// 连接打开
ws.onopen = () => {
  console.log('连接已建立');
  ws.send(JSON.stringify({
    type: 'join',
    room: 'frontend',
    user: 'Tom'
  }));
};

// 接收消息
ws.onmessage = (event) => {
  // event.data 可能是 string / Blob / ArrayBuffer
  const data = JSON.parse(event.data);
  console.log('收到消息:', data);
};

// 连接关闭
ws.onclose = (event) => {
  console.log('连接关闭:', event.code, event.reason);
  // 常见 code: 1000=正常关闭, 1001=离开, 1006=异常关闭
  // 自动重连逻辑
  if (!event.wasClean) {
    setTimeout(() => reconnect(), 3000);
  }
};

// 连接错误
ws.onerror = (error) => {
  console.error('WebSocket 错误:', error);
};

// 主动关闭
ws.close(1000, '用户主动离开');

// 连接状态
ws.readyState;
// 0 = CONNECTING（连接中）
// 1 = OPEN（已连接）
// 2 = CLOSING（关闭中）
// 3 = CLOSED（已关闭）
```

**生产环境最佳实践：**
```javascript
class ReconnectWebSocket {
  constructor(url, options = {}) {
    this.url = url;
    this.maxRetries = options.maxRetries ?? Infinity;
    this.reconnectInterval = options.reconnectInterval ?? 3000;
    this.maxReconnectInterval = options.maxReconnectInterval ?? 30000;
    this.retries = 0;
    this.heartbeatInterval = options.heartbeatInterval ?? 30000;
    this.ws = null;
    this.connect();
  }

  connect() {
    this.ws = new WebSocket(this.url);

    this.ws.onopen = () => {
      console.log('WebSocket 已连接');
      this.retries = 0;
      this.startHeartbeat();
    };

    this.ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      if (data.type === 'pong') return; // 心跳回复，忽略
      this.onMessage?.(data);
    };

    this.ws.onclose = () => {
      this.stopHeartbeat();
      this.tryReconnect();
    };

    this.ws.onerror = () => {};
  }

  // 心跳保活：防止连接因空闲被中间设备断开
  startHeartbeat() {
    this.heartbeatTimer = setInterval(() => {
      if (this.ws?.readyState === WebSocket.OPEN) {
        this.ws.send(JSON.stringify({ type: 'ping' }));
      }
    }, this.heartbeatInterval);
  }

  stopHeartbeat() {
    clearInterval(this.heartbeatTimer);
  }

  // 断线重连：指数退避
  tryReconnect() {
    if (this.retries >= this.maxRetries) return;
    const delay = Math.min(
      this.reconnectInterval * Math.pow(2, this.retries),
      this.maxReconnectInterval
    );
    this.retries++;
    console.log(`${delay}ms 后第 ${this.retries} 次重连...`);
    setTimeout(() => this.connect(), delay);
  }

  send(data) {
    if (this.ws?.readyState === WebSocket.OPEN) {
      this.ws.send(typeof data === 'string' ? data : JSON.stringify(data));
    }
  }
}
```

**实际应用：** 即时聊天、实时协作编辑、股票行情推送、多人游戏、实时通知、IoT 设备数据

### 4. Drag & Drop（拖拽 API）

**是什么：** HTML5 原生的拖拽 API，可以让任何元素可拖拽，并在拖拽过程中监听各个阶段的事件。

**拖拽事件生命周期：**
```
被拖拽元素（Drag Source）:
  dragstart  → 开始拖拽（鼠标按下并移动）
  drag       → 拖拽过程中持续触发
  dragend    → 拖拽结束（松开鼠标或按 ESC）

放置目标（Drop Target）:
  dragenter  → 拖拽元素进入目标区域
  dragover   → 拖拽元素在目标区域上方（持续触发，必须 preventDefault 才能 drop）
  dragleave  → 拖拽元素离开目标区域
  drop       → 在目标区域释放
```

**基础示例：**
```html
<div id="draggable" draggable="true">拖我</div>
<div id="dropzone">放这里</div>

<script>
const draggable = document.getElementById('draggable');
const dropzone = document.getElementById('dropzone');

// ===== 被拖拽元素 =====
draggable.addEventListener('dragstart', (e) => {
  // 设置拖拽数据（必须设置，否则无法拖拽）
  e.dataTransfer.setData('text/plain', e.target.id);
  e.dataTransfer.setData('application/json', JSON.stringify({ id: 123, name: 'Tom' }));
  // 设置拖拽时的视觉效果
  e.dataTransfer.effectAllowed = 'move';
  // 半透明效果
  e.target.style.opacity = '0.5';
});

draggable.addEventListener('dragend', (e) => {
  e.target.style.opacity = '1';
});

// ===== 放置目标 =====
// dragover 必须阻止默认行为，否则 drop 事件不会触发！
dropzone.addEventListener('dragover', (e) => {
  e.preventDefault(); // ← 关键！
  e.dataTransfer.dropEffect = 'move';
  dropzone.classList.add('drag-over'); // 添加高亮样式
});

dropzone.addEventListener('dragleave', (e) => {
  dropzone.classList.remove('drag-over');
});

dropzone.addEventListener('drop', (e) => {
  e.preventDefault();
  const id = e.dataTransfer.getData('text/plain');
  const jsonData = e.dataTransfer.getData('application/json');
  const draggedElement = document.getElementById(id);
  dropzone.appendChild(draggedElement); // 把元素移到目标区域
  dropzone.classList.remove('drag-over');
});
</script>
```

**DataTransfer 对象核心属性：**
```javascript
e.dataTransfer.setData(format, data)      // 设置拖拽数据
e.dataTransfer.getData(format)             // 获取拖拽数据
e.dataTransfer.effectAllowed               // 允许的操作: 'none' | 'copy' | 'move' | 'link' | 'all'
e.dataTransfer.dropEffect                  // 当前操作类型
e.dataTransfer.files                       // 拖入的文件列表（用于文件上传）
e.dataTransfer.setDragImage(element, x, y) // 自定义拖拽预览图
```

**文件拖拽上传：**
```javascript
const dropzone = document.getElementById('dropzone');

dropzone.addEventListener('dragover', (e) => {
  e.preventDefault();
  e.dataTransfer.dropEffect = 'copy';
});

dropzone.addEventListener('drop', async (e) => {
  e.preventDefault();
  const files = e.dataTransfer.files; // FileList

  for (const file of files) {
    console.log(`文件名: ${file.name}`);
    console.log(`大小: ${file.size} bytes`);
    console.log(`类型: ${file.type}`);

    // 上传文件
    const formData = new FormData();
    formData.append('file', file);
    await fetch('/api/upload', { method: 'POST', body: formData });
  }
});
```

**实际应用：** 看板拖拽排序（Trello）、文件上传、拖拽排序列表、可视化编辑器、页面构建器

### 5. Web Speech API（语音）

**是什么：** 浏览器提供的语音能力 API，包含两个部分：
- **Speech Recognition（语音识别）：** 麦克风录音 → 转文字（STT）
- **Speech Synthesis（语音合成）：** 文字 → 语音播报（TTS）

#### 5.1 语音识别（Speech Recognition）
```javascript
// 兼容性处理
const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition;

if (!SpeechRecognition) {
  console.error('当前浏览器不支持语音识别');
}

const recognition = new SpeechRecognition();

// 配置
recognition.lang = 'zh-CN';          // 语言：中文
recognition.continuous = true;        // 持续识别（不停顿）
recognition.interimResults = true;    // 返回临时结果（边说边出文字）
recognition.maxAlternatives = 1;      // 每个结果最多返回几个候选

// 开始识别
recognition.start();

// 获得结果
recognition.onresult = (event) => {
  // event.results 是 SpeechRecognitionResultList
  for (let i = event.resultIndex; i < event.results.length; i++) {
    const result = event.results[i];
    const transcript = result[0].transcript;        // 识别的文字
    const confidence = result[0].confidence;         // 置信度 0-1
    const isFinal = result.isFinal;                  // 是否是最终结果

    if (isFinal) {
      console.log(`最终结果: "${transcript}" (置信度: ${confidence})`);
    } else {
      console.log(`临时结果: "${transcript}"`);
    }
  }
};

// 识别结束
recognition.onend = () => {
  console.log('识别结束');
  // 如果需要持续识别，需要手动重启
  // recognition.start();
};

// 错误处理
recognition.onerror = (event) => {
  switch (event.error) {
    case 'not-allowed':     // 用户拒绝了麦克风权限
      console.error('请允许麦克风权限');
      break;
    case 'no-speech':       // 没有检测到语音
      console.log('没有检测到语音');
      break;
    case 'network':         // 网络错误
      console.error('网络错误');
      break;
    case 'aborted':         // 被中止
      break;
  }
};

// 停止识别
recognition.stop();   // 停止并返回已收集的结果
recognition.abort();  // 立即中止，丢弃结果
```

#### 5.2 语音合成（Speech Synthesis）
```javascript
// 获取可用的语音列表（异步加载）
window.speechSynthesis.onvoiceschanged = () => {
  const voices = window.speechSynthesis.getVoices();
  voices.forEach(voice => {
    console.log(`${voice.name} (${voice.lang}) ${voice.default ? '— 默认' : ''}`);
    // 如: "Microsoft Yaoyao — Chinese (zh-CN)"
    //     "Google 普通话 — zh-CN"
    //     "Samantha — en-US"
  });
};

// 创建语音合成实例
const utterance = new SpeechSynthesisUtterance('你好，欢迎使用语音合成功能');

// 配置
utterance.lang = 'zh-CN';            // 语言
utterance.rate = 1.0;                 // 语速：0.1 - 10（默认 1）
utterance.pitch = 1.0;                // 音调：0 - 2（默认 1）
utterance.volume = 1.0;               // 音量：0 - 1（默认 1）
utterance.voice = speechSynthesis.getVoices().find(v => v.lang === 'zh-CN');

// 事件监听
utterance.onstart = () => console.log('开始播放');
utterance.onend = () => console.log('播放完成');
utterance.onpause = () => console.log('暂停');
utterance.onresume = () => console.log('继续');
utterance.onerror = (e) => console.error('播放错误:', e.error);

// 控制
window.speechSynthesis.speak(utterance);    // 播放
window.speechSynthesis.pause();             // 暂停
window.speechSynthesis.resume();            // 继续
window.speechSynthesis.cancel();            // 取消（停止所有）
window.speechSynthesis.speaking;            // 是否正在播放
window.speechSynthesis.pending;             // 队列中是否还有待播放的
```

**注意事项：**
- 语音识别需要 HTTPS + 用户授权麦克风
- Chrome 的语音识别依赖 Google 服务器（需要网络）
- 语音合成在部分浏览器中长时间文本会被截断，需要分段播放
- Safari 兼容性需要注意（使用 `webkit` 前缀）

**实际应用：** 语音输入/搜索、语音助手、无障碍辅助（读屏）、多语言学习、语音转字幕

## 三、CSS

### 1. 盒模型
- **标准盒模型**（`box-sizing: content-box`）：`width` = content
- **IE 盒模型**（`box-sizing: border-box`）：`width` = content + padding + border
- 实际开发推荐 `border-box`

### 2. BFC（块级格式化上下文）
**触发条件：**
- `overflow: hidden/auto/scroll`
- `display: flex/grid/inline-block/table-cell`
- `position: absolute/fixed`
- `float` 不为 none

**作用：**
- 阻止外边距折叠
- 包含浮动元素（清除浮动）
- 阻止元素被浮动元素覆盖

### 3. Flex 布局核心
```css
.container {
  display: flex;
  justify-content: center; /* 主轴 */
  align-items: center;     /* 交叉轴 */
  flex-direction: row | column;
  flex-wrap: wrap;
  gap: 10px;
}
.item {
  flex: 1; /* flex-grow: 1, flex-shrink: 1, flex-basis: 0% */
}
```

### 4. 水平垂直居中方案
```css
/* 方案1: Flex */
.parent { display: flex; justify-content: center; align-items: center; }

/* 方案2: Grid */
.parent { display: grid; place-items: center; }

/* 方案3: 定位 + transform */
.child { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); }

/* 方案4: 定位 + margin auto */
.child { position: absolute; inset: 0; margin: auto; }
```

### 5. CSS 选择器优先级
```
!important > 内联样式(1000) > ID(100) > 类/伪类/属性(10) > 标签/伪元素(1) > 通配符(0)
```

### 6. CSS 动画
```css
/* transition: 状态变化过渡 */
.element { transition: all 0.3s ease; }

/* animation: 关键帧动画 */
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}
.element { animation: fadeIn 0.3s ease forwards; }
```
- **GPU 加速属性**：`transform` `opacity`（不触发重排重绘）
- **避免动画属性**：`width` `height` `top` `left`（触发重排）

### 7. 响应式方案
- **媒体查询**：`@media (max-width: 768px) {}`
- **rem + 根字体大小**：`html { font-size: calc(100vw / 375 * 16); }`
- **vw/vh**：视口单位
- **Container Query**（新）：基于容器而非视口
- **clamp()**：`font-size: clamp(14px, 2vw, 18px)`

## 四、JavaScript 核心

### 1. 数据类型
**基本类型（7种）：** `string` `number` `boolean` `null` `undefined` `symbol` `bigint`

**引用类型：** `Object` `Array` `Function` `Date` `RegExp` `Map` `Set`

**类型判断：**
```javascript
typeof null           // "object" (历史遗留bug)
typeof undefined      // "undefined"
typeof function(){}   // "function"
typeof []             // "object"

// 精确判断
Object.prototype.toString.call([])      // "[object Array]"
Array.isArray([])                       // true
value instanceof Array                  // true（跨 iframe 失效）
```

### 2. 闭包
**定义：** 函数能够访问其词法作用域中变量的能力，即使函数在其词法作用域之外执行。

```javascript
function createCounter() {
  let count = 0; // 被闭包捕获
  return {
    increment: () => ++count,
    getCount: () => count,
  };
}
```

**应用场景：** 防抖节流、柯里化、模块模式、私有变量
**注意：** 闭包会阻止变量被回收，可能导致内存泄漏

### 3. 原型链
```
实例 → __proto__ → 构造函数.prototype → __proto__ → Object.prototype → __proto__ → null
```

```javascript
function Person(name) { this.name = name; }
Person.prototype.sayHi = function() { console.log(this.name); };

const p = new Person('Tom');
p.hasOwnProperty('name');  // true
p instanceof Person;       // true
```

**new 操作符做了什么？**
1. 创建空对象
2. 设置原型：`obj.__proto__ = Constructor.prototype`
3. 绑定 this 执行构造函数
4. 返回对象（如果构造函数返回对象则返回该对象）

### 4. this 指向
| 调用方式 | this 指向 |
|---------|----------|
| 普通函数调用 | 全局对象（严格模式下 undefined） |
| 对象方法调用 | 调用的对象 |
| `new` 调用 | 新创建的实例 |
| `call/apply/bind` | 手动指定的对象 |
| 箭头函数 | 定义时外层 this（不可改变） |

### 5. 事件循环（Event Loop）
```
┌───────────────────────┐
│     Call Stack         │  同步代码执行
└───────────┬───────────┘
            │
┌───────────▼───────────┐
│   Microtask Queue      │  Promise.then / MutationObserver / queueMicrotask
└───────────┬───────────┘
            │
┌───────────▼───────────┐
│   Macrotask Queue      │  setTimeout / setInterval / I/O / UI 渲染
└───────────────────────┘
```

**执行顺序：** 同步代码 → 清空微任务 → 一个宏任务 → 清空微任务 → ...

```javascript
console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// 输出: 1, 4, 3, 2
```

### 6. Promise
```javascript
// 三种状态: pending → fulfilled / pending → rejected（不可逆）

// 手写 Promise 核心思路
class MyPromise {
  constructor(executor) {
    this.status = 'pending';
    this.value = undefined;
    this.reason = undefined;
    this.onFulfilledCallbacks = [];
    this.onRejectedCallbacks = [];

    const resolve = (value) => {
      if (this.status === 'pending') {
        this.status = 'fulfilled';
        this.value = value;
        this.onFulfilledCallbacks.forEach(fn => fn());
      }
    };

    const reject = (reason) => {
      if (this.status === 'pending') {
        this.status = 'rejected';
        this.reason = reason;
        this.onRejectedCallbacks.forEach(fn => fn());
      }
    };

    try { executor(resolve, reject); }
    catch (e) { reject(e); }
  }

  then(onFulfilled, onRejected) {
    onFulfilled = typeof onFulfilled === 'function' ? onFulfilled : v => v;
    onRejected = typeof onRejected === 'function' ? onRejected : e => { throw e; };

    const promise2 = new MyPromise((resolve, reject) => {
      if (this.status === 'fulfilled') {
        queueMicrotask(() => {
          try { resolve(onFulfilled(this.value)); }
          catch (e) { reject(e); }
        });
      }
      if (this.status === 'rejected') {
        queueMicrotask(() => {
          try { resolve(onRejected(this.reason)); }
          catch (e) { reject(e); }
        });
      }
      if (this.status === 'pending') {
        this.onFulfilledCallbacks.push(() => {
          queueMicrotask(() => {
            try { resolve(onFulfilled(this.value)); }
            catch (e) { reject(e); }
          });
        });
        this.onRejectedCallbacks.push(() => {
          queueMicrotask(() => {
            try { resolve(onRejected(this.reason)); }
            catch (e) { reject(e); }
          });
        });
      }
    });
    return promise2;
  }
}
```

### 8. 深拷贝
```javascript
function deepClone(obj, map = new WeakMap()) {
  if (obj === null || typeof obj !== 'object') return obj;
  if (map.has(obj)) return map.get(obj); // 解决循环引用

  const clone = Array.isArray(obj) ? [] : {};
  map.set(obj, clone);

  for (const key of Reflect.ownKeys(obj)) {
    clone[key] = deepClone(obj[key], map);
  }
  return clone;
}
```

### 9. 防抖与节流
```javascript
// 防抖: 事件停止触发 n 秒后执行一次
function debounce(fn, delay) {
  let timer;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流: n 秒内只执行一次
function throttle(fn, interval) {
  let lastTime = 0;
  return function (...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}
```
